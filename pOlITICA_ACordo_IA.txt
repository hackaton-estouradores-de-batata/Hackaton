POLITICA DE ACORDO E DEFESA - V5
ESPECIFICACAO RESUMIDA PARA IMPLEMENTACAO POR IA

1. OBJETIVO

Implementar um algoritmo que receba os dados de um processo e devolva:
- decisao final: ACORDO ou DEFESA
- p_suc
- p_per
- vpc
- vej
- abertura
- alvo
- teto
- justificativa curta

Regra economica central:
- acordo so eh bom quando o valor negociado fica abaixo do custo esperado de defender
- custo esperado de defender = VEJ


2. ENTRADAS OBRIGATORIAS

- PED: valor do pedido
- UF: estado do processo
- SUB: generico, golpe ou indefinido
- docs_presentes:
  - contrato
  - comprovante_credito
  - extrato
  - demonstrativo_evolucao_divida
  - dossie
  - laudo_referenciado


3. CONTAGEM DE DOCUMENTOS

Cada documento valido soma 1 ponto.

QTD_DOCS = soma dos 6 documentos validos

Faixa:
- minimo = 0
- maximo = 6

Regras:
- documento duplicado nao soma
- documento inutilizavel nao soma
- documento contraditorio deve gerar flag para revisao humana


4. FONTES DE VERDADE

Use os valores numericos exatos da V5 canonica em:
- desconto por UF
- MATRIZ_GERAL_6D
- MATRIZ_GENERICO_6D
- MATRIZ_GOLPE_6D

Arquivo canonico:
- politica_acordos_pipelineV5.txt

Regra de escolha da matriz:
- se SUB = generico -> usar MATRIZ_GENERICO_6D
- se SUB = golpe -> usar MATRIZ_GOLPE_6D
- se SUB estiver ausente, indefinido ou nao mapeado -> usar MATRIZ_GERAL_6D

Fallback de UF:
- se UF nao existir na matriz escolhida -> usar linha TOTAL
- se UF nao existir na tabela de desconto -> usar desconto_uf = 0.30


5. VARIAVEIS DO MODELO

- P_CEL_D: taxa da UF para a quantidade D na matriz escolhida
- P_TOTAL_D: taxa TOTAL para a quantidade D na matriz escolhida
- P_UF_TOTAL: coluna total da UF na matriz escolhida
- P_SUB_TOTAL: coluna total da linha TOTAL na matriz escolhida
- W_D: peso da UF na quantidade D
- P_D_RAW: taxa preliminar antes da correcao monotona
- P_D_CORR: taxa final corrigida
- P_SUC: probabilidade final de sucesso
- P_PER: probabilidade final de perda
- DESC_UF: desconto medio da UF quando perde
- F_PAG: percentual medio efetivamente pago se perder
- VPC: valor provavel de condenacao
- VEJ: valor esperado judicial
- ABERTURA: primeira proposta
- ALVO: melhor valor de fechamento para o banco
- TETO: maior valor que ainda eh melhor do que defender
- TETO_PCT: TETO / PED


6. PESOS POR QUANTIDADE DE DOCUMENTOS

Usar exatamente:
- W_0 = 0.05
- W_1 = 0.15
- W_2 = 0.30
- W_3 = 0.45
- W_4 = 0.60
- W_5 = 0.75
- W_6 = 0.90

Interpretacao:
- com pouca prova, a media nacional pesa mais
- com muita prova, a UF pesa mais


7. FORMULA PARA 1 A 6 DOCUMENTOS

Para cada D em 1..6:

P_D_RAW = P_TOTAL_D + W_D * (P_CEL_D - P_TOTAL_D)

Interpretacao:
- partir da media nacional daquela quantidade de documentos
- puxar parcialmente em direcao ao historico da UF


8. REGRA ESPECIAL PARA 0 DOCUMENTOS

Nao usar o valor cru de 0 docs da UF diretamente.

Calcular:

P1_PROXY = P_TOTAL_1 + W_1 * (P_CEL_1 - P_TOTAL_1)
GAP0 = P_TOTAL_1 - P_TOTAL_0
UF_SHIFT = P_UF_TOTAL - P_SUB_TOTAL
P0_CONTEXT = P_TOTAL_0 + 0.15 * UF_SHIFT
P_0_RAW = clamp( min(P0_CONTEXT, P1_PROXY - GAP0), 0, P1_PROXY )

Interpretacao:
- 0 docs deve ser conservador
- 0 docs nunca pode ficar acima de 1 doc
- a UF entra so como pequeno ajuste de contexto


9. CORRECAO MONOTONA OBRIGATORIA

Depois de calcular P_0_RAW, P_1_RAW, ..., P_6_RAW:

P_0_CORR = clamp(P_0_RAW, 0, 0.98)
P_1_CORR = clamp(max(P_1_RAW, P_0_CORR), 0, 0.98)
P_2_CORR = clamp(max(P_2_RAW, P_1_CORR), 0, 0.98)
P_3_CORR = clamp(max(P_3_RAW, P_2_CORR), 0, 0.98)
P_4_CORR = clamp(max(P_4_RAW, P_3_CORR), 0, 0.98)
P_5_CORR = clamp(max(P_5_RAW, P_4_CORR), 0, 0.98)
P_6_CORR = clamp(max(P_6_RAW, P_5_CORR), 0, 0.98)

Invariante obrigatoria:
- P(0) <= P(1) <= P(2) <= P(3) <= P(4) <= P(5) <= P(6)


10. PISO E TETO TECNICOS

P_SUC = clamp(P_QTD_DOCS_CORR, 0.01, 0.98)
P_PER = 1 - P_SUC

Interpretacao:
- nunca afirmar 0% absoluto
- nunca afirmar 100% absoluto


11. CONTA ECONOMICA

F_PAG = 1 - DESC_UF
VPC = PED * F_PAG
VEJ = VPC * P_PER

Interpretacao:
- VPC = quanto o banco costuma pagar se perder
- VEJ = quanto custa defender, em valor esperado


12. ESCADA DE ACORDO

F_AB = clamp(0.90 - 0.05 * QTD_DOCS, 0.60, 0.95)
F_AL = clamp(F_AB + 0.08, 0.65, 0.97)
F_TET = clamp(F_AB + 0.15, 0.70, 0.99)

ABERTURA = arredondar_100(VEJ * F_AB)
ALVO = arredondar_100(VEJ * F_AL)
TETO = arredondar_100(VEJ * F_TET)

Interpretacao:
- pouca prova -> proposta pode ficar mais perto do VEJ
- muita prova -> proposta deve ficar mais longe do VEJ
- ALVO eh o valor recomendado para fechar


13. REGRA DE DECISAO FINAL

TETO_PCT = TETO / PED

Se TETO_PCT >= 0.25:
- decisao = ACORDO

Se TETO_PCT < 0.25:
- decisao = DEFESA

Interpretacao:
- acordo so deve ser recomendado quando ainda existir teto racional e
  comercialmente viavel
- abaixo de 25% do pedido, a defesa tende a ser superior


14. SAIDA ESPERADA

O algoritmo deve devolver:
- decisao
- p_suc
- p_per
- vpc
- vej
- abertura
- alvo
- teto
- justificativa

Formato recomendado da justificativa:
- "ACORDO porque o ALVO fica abaixo do VEJ em R$ X"
- "DEFESA porque o TETO racional fica abaixo do piso comercial"


15. PSEUDOCODIGO MINIMO

entrada:
- PED
- UF
- SUB
- docs_presentes

qtd_docs = contar_docs_validos(docs_presentes)

if SUB == "generico":
    mat = MATRIZ_GENERICO_6D
elseif SUB == "golpe":
    mat = MATRIZ_GOLPE_6D
else:
    mat = MATRIZ_GERAL_6D

if UF nao existir em mat:
    uf_key = "TOTAL"
else:
    uf_key = UF

if UF existir na tabela de desconto:
    desconto_uf = desconto[UF]
else:
    desconto_uf = 0.30

para d de 1 ate 6:
    p_raw[d] = mat["TOTAL"][d] + peso[d] * (mat[uf_key][d] - mat["TOTAL"][d])

gap0 = mat["TOTAL"][1] - mat["TOTAL"][0]
uf_shift = mat[uf_key]["T"] - mat["TOTAL"]["T"]
p1_proxy = p_raw[1]
p0_context = mat["TOTAL"][0] + 0.15 * uf_shift
p_raw[0] = limitar(minimo(p0_context, p1_proxy - gap0), 0, p1_proxy)

p_corr[0] = limitar(p_raw[0], 0, 0.98)
para d de 1 ate 6:
    p_corr[d] = limitar(maximo(p_raw[d], p_corr[d - 1]), 0, 0.98)

p_suc = limitar(p_corr[qtd_docs], 0.01, 0.98)
p_per = 1 - p_suc

f_pag = 1 - desconto_uf
vpc = PED * f_pag
vej = vpc * p_per

f_ab = limitar(0.90 - 0.05 * qtd_docs, 0.60, 0.95)
f_al = limitar(f_ab + 0.08, 0.65, 0.97)
f_tet = limitar(f_ab + 0.15, 0.70, 0.99)

abertura = arredondar_100(vej * f_ab)
alvo = arredondar_100(vej * f_al)
teto = arredondar_100(vej * f_tet)
teto_pct = teto / PED

if teto_pct >= 0.25:
    decisao = "ACORDO"
else:
    decisao = "DEFESA"

saida:
- decisao
- p_suc
- p_per
- vpc
- vej
- abertura
- alvo
- teto


16. INVARIANTES QUE NAO PODEM SER VIOLADAS

- usar matriz por subassunto antes de qualquer conta
- contar 6 documentos, nao 5
- nao usar 0 docs cru da UF diretamente
- garantir monotonicidade
- aplicar piso de 1% e teto de 98% em p_suc
- calcular acordo sempre a partir do VEJ
- nunca recomendar acordo acima do TETO
- usar fallback TOTAL para RR ou UF sem historico


17. ERROS COMUNS A EVITAR

- usar a matriz geral no lugar da matriz do subassunto
- somar subassunto como ajuste extra depois de escolher a matriz
- usar 0 docs cru diretamente
- deixar 1 doc menor que 0 docs ou 6 docs menor que 5 docs
- calcular acordo como percentual fixo do pedido
- esquecer de arredondar ABERTURA, ALVO e TETO para R$ 100
- esquecer fallback de RR


18. RESUMO EXECUTIVO PARA O AGENTE

Pense assim:
- primeiro escolha a matriz certa
- depois estime a chance de sucesso pela quantidade de documentos
- depois converta isso em custo esperado da defesa
- depois gere a escada de acordo a partir do VEJ
- por fim decida: se o teto racional ainda for >= 25% do pedido, recomende ACORDO; se nao, recomende DEFESA
